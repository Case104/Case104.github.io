Arrays and Hashes

January 3, 2016

Storing data

Arrays and hashes both serve as mediums to store data. More spefically, they alllow you to store a collection of data including variables, strings, integers, floats, and other objects including arrays and hashes. This is an important function because you will come across many instances where you will need to be able access

Arrays

Arrays are defined by assigning elements to a variable. One way to this would be to type array = [1, 2, 3]. Again, you can also assign other types of data to an array, like so array = [1, "two", 3.0]. Once you have an array defined, you can add to it using the .push or <<. You can remove the last element of the array using the .pop method.

To access the data in an array, you would call its index. An array's index starts at 0, and increases by one for each element. Lets explore the concept with this sample array.

array = [1, 2, 3, 4] // array[0] => 1 // array[1] => 2
As you can see, accessing 0 returns the first piece of data in the array, and it progresses linearly from there.

Hashes

Hashes store their data using key => value pairs. An example of their syntax is as follows.

my_hash = {"key" => "value", "name" => "Jonathan", "age" => 24}
The data is not stored in any particular order, and can be accesses by referencing the key.my_hash["key"] would return the value"value".

Comparing the two

First and foremost, arrays and hashes both have their place as a tool to store your collection of data. Where they differ is in their methods of retreiving the information. Arrays are always ordered, and the data is stored in a logical progression. Hashes on the other hand, have no specified order, and are more often used when the key and value are easily associated.
