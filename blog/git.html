Git Workflow for Pushing Content

December 6, 2015

Git and Github, understanding the difference

Git and GitHub are both tools used to store and share your work, as well as collaborate and work as a team. After learning that JavaScript and Java have nothing in common, I approached Git and Github as seperate entities at first, but quickly learned that they do work together to provide effective data sharing and version control.

I'm still working out the details, but it is my understanding that Git was origionally created as a tool for version control. Git lives locally on your computer, and stores data like a snapshot. Each commit that you make acts as a "savepoint" that you can return to. Files reside in three states, commited, modified, and staged. Committed means that the data is stored locally. Modified means that changes have been made, but nothing has been saved yet. Staged means that you have marked a file to go into your next committed snapshot.
For more, and better written information, check out this link.

Github on the other hand was created to allow sharing of information, and storage remotely, outside of your computer. It works with Git in how it stores and shares information, using the same snapshot model. I think the biggest benefit of Github is that many people can be working in seperate "branches" that have no effect on the source code while they are working. Once they are done, it goes through a sort of peer review called a pull request. This is great because it allows developers to work side by side in a stable environment, while contributing work as well as checking over each other's code.

Saving your work and pushing it to GitHub, the Workflow

I'm still learning the "how", but so far this workflow has been working for me to push my code from my local machine to GitHub.
1. Check your status, current branch, and run a pull before doing anything. This makes sure that you are going into your new work clean. Do this by typing "git status", "git branch", and "git pull".
2. Create a new branch. "git checkout -b branch-name".
3. Create your code.
4. Add your work to be committed. "git add work.extension"
5. Commit your work. "git commit -m "your comment here""
6. Push your work to github. Log in if necessary. "git push origin branch-name"
7. Complete your pull request on GitHub.
8. Move back to your master branch. "git checkout master"
9. Pull the information back to your computer. "git fetch origin master"
10. Merge them. "git merge origin/master"
11. Delete your pulled branches. "git branch -d branch-name"
